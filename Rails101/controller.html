Controller and Views

Generate a Controller

You can also create these files YourSelf but this is more conventional.By convention Controllers are Plural and Models are Singular.


    $ rails generate controller Posts
    create  app/controllers/posts_controller.rb
    invoke  erb
    create    app/views/posts
    invoke  test_unit
    create    test/controllers/posts_controller_test.rb
    invoke  helper
    create    app/helpers/posts_helper.rb
    invoke    test_unit
    invoke  assets
    invoke    coffee
    create      app/assets/javascripts/posts.coffee
    invoke    scss
    create      app/assets/stylesheets/posts.scss

Adding a new Page

Edit the Controller

You should now add the index method to posts_controller.rb


    class PostsController < ApplicationController
      def index

      end
    end

Add the corresponding view

You should now make a new file "index.html.erb" in app/views/posts


      <h1>Hello World!</h1>

Modifying the Routes

You should now make an entry in config/routes.rb to override the default Rails page with the one we just created


        Rails.application.routes.draw do
          root 'posts#index'
        end

This tells rails to look at the posts controller and in that the index method and the corresponding view

Go back to the Browser and reload http://localhost:3000


Adding another Controller

Add a Controller for static Pages

This can be used to show the about page, team page or contact page etc


  $ rails g controller Pages
        create  app/controllers/pages_controller.rb
        invoke  erb
        create    app/views/pages
        invoke  test_unit
        create    test/controllers/pages_controller_test.rb
        invoke  helper
        create    app/helpers/pages_helper.rb
        invoke    test_unit
        invoke  assets
        invoke    coffee
        create      app/assets/javascripts/pages.coffee
        invoke    scss
        create      app/assets/stylesheets/pages.scss

Edit the Controller

You should now add the index method to pages_controller.rb


  class PostsController < ApplicationController
    def about

    end
  end

Add the corresponding view

You should now make a new file "about.html.erb" in app/views/posts


    <h1>About!</h1>

Modifying the Routes

You should now make an entry in config/routes.rb to override the default Rails page with the one we just created


      Rails.application.routes.draw do
        root 'posts#index'
        get 'about' => 'pages#about'
      end

This tells rails to look at the pages controller but get the about page directly and not /pages/about

Go back to the Browser and reload http://localhost:3000


Adding some Dynamic Content

Add the Title Field to the controller

Edit pages_controller.rb


    class PagesController < ApplicationController
      def about
        @title = 'About Us'
      end
    end

Edit the View to reflect Changes

Data can be sent from controller to the view very easily


    
<%= @title %>

Add the Content Section

Edit pages_controller.rb


        class PagesController < ApplicationController
          def about
            @title = 'About Us'
            @content = 'This is the About Page.'
          end
        end

Showing the Content on the View


        
<%= @title %>

        
<%= @content %>

Adding a resource

Close the Pages Controller

We will not be working with that

But for our Posts we need a lot of methods like :index(which we have) a show view and a show method, to show the individual posts and index will will be a list of Posts new, edit, destroy

Instead of doing that manually we can Claim Posts to be a resource


              Rails.application.routes.draw do
                root 'posts#index'
                get 'about' => 'pages#about'
                resources :posts
              end

You can see the routes that this added


      $ rake routes
         Prefix Verb   URI Pattern               Controller#Action
           root GET    /                         posts#index
          about GET    /about(.:format)          pages#about
          posts GET    /posts(.:format)          posts#index
                POST   /posts(.:format)          posts#create
       new_post GET    /posts/new(.:format)      posts#new
      edit_post GET    /posts/:id/edit(.:format) posts#edit
           post GET    /posts/:id(.:format)      posts#show
                PATCH  /posts/:id(.:format)      posts#update
                PUT    /posts/:id(.:format)      posts#update
                DELETE /posts/:id(.:format)      posts#destroy
