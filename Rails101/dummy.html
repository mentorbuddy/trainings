Getting Started

Dive into Ruby on Rails, a powerful Framework for building Web Applications using the Ruby Programming Language.

Download

To start using Rails and get a standard Environment, You must use only the steps that are mentioned here. This Guide assumes a Windows Development Environment

Rails Installer

This is the Rails 4.2.5 version that comes with a stable Rails, and stes up Git, SQLite etc as well. This option requires little to no setup. Use this if you are unfamiliar with Rails.

Railsfile_download

Heroku Toolbelt

This is Heroku ToolBelt, this will allow us to host the application. By choosing Heroku as our staging server as well, we have more control over which components to include in future releases.

Herokufile_download

Your Favourite Text Editor

This is an optional step. Any Text Editor that doesnot modify the source initself would do. Most widely used are Atom.io, Sublime Text etc.

Atomfile_download

Node-js

This should take care of most JavaScript related errors. It's best known as a popular means for JavaScript coders to build real-time Web APIs

Node-JSfile_download

Checking Installations

Verfiy the Installations for a Basic Rails Setup.


    $ ruby --version

Setup

Start a New Application

You can now setup a new application and start using Rails


  $ rails new mysite
  $ cd mysite

Project Structure

After executing the Above Steps, open the files into the directory where your website is located. Your directory will look something like this.

You'll notice that there are a few more folders but these are mostly the Files that we will Focus on during this session.

Starting up the Web Server

You actually have a functional Rails application already. To see it, you need to start a web server on your development machine. You can do this by running the following in the mysite directory:


  $ rails server

Third-party Options

There are a few community-made options for you to easily include Materialize in your project. Keep in mind these are untested and may be out-of-date.

Ruby Gem

See here for documentation on this gem.


  gem 'materialize-sass'

Meteor Package


  meteor add materialize:materialize

Ember Package


  # install via npm
  $ npm install ember-cli-materialize --save-dev
  # make ember-cli fetch internal dependencies
  $ ember g ember-cli-materialize

Sass Setup

This section is only relevant if you chose to download the Sass version of Materialize.

Compiling Sass

Instead of having a css folder, you will find that the download instead contains many .scss files which contain the styles of individual components. Unfortunately, the browser cannot interpret Sass, so you must have your Sass compiler compile the scss/materialize.scss into a regular CSS file. At this point you can link this newly outputted file in your HTML page.


  MyWebsite/
  |--css/
  |  |--materialize.css <-- compiled from scss/materialize.scss
  |
  |--fonts/
  |  |--roboto/
  |
  |--js/
  |  |--materialize.js
  |
  |--scss/
  |  |--materialize.scss
  |  |--components/
  |
  |--index.html
